Waterfall Methodology

Waterfall Cycle

Key terms

  • Linear/sequential
  • Documentation

Model


The waterfall Model starts with:

  1. Determining the requirements-This is where documenting requirements gathered from questionnaires, observation and interviews. Acceptance tests are often paired with requirements.
  2. Analysis-Feasibility TELOS is asked; T=technical such as is the technology available?,E=economical is it cost effective?,L=Legal is it upholing the laws? O=Operations such as how effective the project will be in solving the problem, S=schedule can it be done in time?.
  3. Design-This includes data dictionary, data flow diagrams, flowcharts and structure diagrams.
  4. Coding-This is where the coding is done for the project
  5. Testing-White/black box, alpha/beta, performance tests, etc.
  6. Maintenance-Corrective, perfective, adaptive maintenance this is done to keep the software product usable in the changing future enviroment.

The advantages of the Waterfall cycle

There is lots of documentation involved in the project
Easy to schedule and arrange tasks which helps keep track of the project
It's simple and easy to understand model

The disadvantages of the Waterfall cycle

Lots of documentation
If in testing the systems fails to meet requirements, either the requirements must be modified, or a substantial change in the design is required. “In effect the development process has returned to the origin and one can expect up to a lO0-percent overrun in schedule and/or costs” (p329, Royce, 1970).
Misunderstanding of requirements discovered too late means that there is more work to go back through and change.
Users not involved until testing
Cannot accommodate changing requirements when needed
No working software is produced until late during the life cycle.
Efficient code is not prioritized

When might it be used?

Straight forward projects where requirements are known and understood at the beginning and don't change.
When there is no benefit to the customer of having parts of the system available early.
Short projects
